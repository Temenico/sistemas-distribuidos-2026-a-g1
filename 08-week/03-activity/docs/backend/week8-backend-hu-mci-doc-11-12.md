# Week 8 - Backend (HU-MCI, HU-doc, HU-BE-11, HU-BE-12)

## Contexto
En Week 8 se consolidó el trabajo de separación y estandarización del backend iniciado en Week 7, priorizando consistencia técnica entre microservicios y trazabilidad documental por historia de usuario.

Repositorio: <https://github.com/jose6668/FarmaExpres_Backend>

## Historias de Usuario Incluidas

### HU-MCI - Migración de Backend de Español a Inglés
- Alcance: renombrado progresivo de componentes para estandarización técnica.
- Resultado: mayor uniformidad en entidades, DTOs, controladores y servicios.
- Impacto: mejora de mantenibilidad y lectura entre equipos.

### HU-doc - Documentación técnica de microservicios
- Alcance: fortalecimiento de documentación en `README.md` y estructura por servicio.
- Resultado: mayor claridad de endpoints, configuración y ejecución local.
- Impacto: onboarding más rápido y menor ambigüedad en integración.

### HU-BE-11 - Ajustes funcionales de dominio
- Alcance: ajustes de comportamiento backend para alinear reglas de negocio.
- Resultado: preparación de servicios para evolución de consultas y reportes.
- Impacto: base más estable para iteraciones funcionales siguientes.

### HU-BE-12 - Campo de stock mínimo en producto
- Alcance: incorporación/normalización del stock mínimo en lógica de inventario.
- Resultado: soporte para alertas y control de niveles críticos.
- Impacto: mejora de trazabilidad operativa para frontend y reportes.

## Estado General de Backend en Week 8
- Se consolidó una base backend más homogénea en nomenclatura y documentación.
- Se sostuvo la separación por microservicios con responsabilidades claras.
- Se alinearon ajustes para mantener coherencia entre autenticación, inventario y gateway.
- El trabajo quedó preparado para continuar en Week 9 con HU orientadas a reportes, filtros y consolidación documental.
